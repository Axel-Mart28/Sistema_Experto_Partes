1. Entorno de Adquisición de Conocimiento (Experto, Cognimático, Sensores)
<br>
<div align="justity">
¿Qué es?
Es el conjunto de fuentes de donde el sistema extrae la información fundamental para su funcionamiento. Incluye a los especialistas humanos (Expertos), los ingenieros de conocimiento (Cognimáticos), e incluso entradas físicas directas (Sensores y Bases de Datos externas).

¿Para qué sirve?
Para dotar al sistema del "saber hacer" especializado y de los datos del mundo real que necesita analizar. Sin esta fase, el sistema no tendría inteligencia ni contexto para operar.

¿Cómo funciona?
El experto humano transfiere su experiencia empírica y teórica al ingeniero de conocimiento (o cognimático). Este último traduce esa información humana a reglas lógicas y algoritmos estructurados. Simultáneamente, los sensores captan señales físicas del entorno para alimentar el sistema con variables en tiempo real.

Ejemplo Práctico:
En una investigación para mejorar la precisión de servomotores MG90S, el Experto es el ingeniero mecatrónico que comprende la dinámica del motor. El Cognimático programa las reglas en Python para el sistema. Los Sensores son los encoders magnéticos de 12 bits que envían lecturas de posición al módulo de adquisición.
